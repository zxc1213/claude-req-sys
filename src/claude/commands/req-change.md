# 需求变更处理命令

处理任务执行过程中的需求变更，评估影响并调整执行计划。

## 用法

\`\`\`bash
/req:change <req-id> [--reason <理由>]
\`\`\`

## 选项

- \`req-id\` - 需求 ID
- \`--reason <理由>\` - 变更理由
- \`--impact\` - 显示影响分析
- \`--type <类型>\` - 指定变更类型

## 变更分类

| 类型     | 特征                   | 处理方式           |
| -------- | ---------------------- | ------------------ |
| 小调整   | 文案、样式微调，<5分钟 | 直接执行           |
| 中等变更 | 影响1-2个任务          | 快速审查后调整     |
| 重大变更 | 改变架构，影响多个任务 | 完整分析后重新规划 |

## 输出示例

### 小调整

\`\`\`markdown
📝 需求变更 - REQ-20260514-001

🔍 变更分类: 小调整
✅ 变更完成，继续执行
\`\`\`

### 重大变更

\`\`\`markdown
🚨 重大需求变更 - REQ-20260514-001

⚠️ 影响评估:
架构变更: 是
影响任务: 8个 (全部)

建议: 重新分析
\`\`\`
