# 需求验证检查命令

在需求管理的各个关键阶段提供系统性的验证检查清单，确保每个阶段的工作质量和完整性。

## 用法

\`\`\`bash
/req:verify <checkpoint> <req-id>
\`\`\`

## 选项

- \`checkpoint\` - 检查点类型
  - \`design\` - 设计验证
  - \`implementation\` - 实现验证
  - \`deployment\` - 部署验证
- \`req-id\` - 需求 ID（如：REQ-20260514-001）

## 检查点

### Checkpoint 1: 设计验证 (design)

**触发时机**：需求设计完成后

### Checkpoint 2: 实现验证 (implementation)

**触发时机**：代码实现完成后

### Checkpoint 3: 部署验证 (deployment)

**触发时机**：准备部署前

## 验证状态

| 状态     | 条件           | 建议                 |
| -------- | -------------- | -------------------- |
| 通过     | 所有检查项完成 | 可以进入下一阶段     |
| 部分通过 | 通过率 ≥ 80%   | 完成关键缺失项后继续 |
| 未通过   | 通过率 < 80%   | 需要全面整改         |

## 相关命令

- \`/req:quality\` - 质量门禁检查
- \`/req:test-plan\` - 生成测试计划
- \`/req\` - 创建需求
