# 文档迁移命令

将旧格式的需求文档安全地迁移到新的统一格式，支持备份和回滚。

## 用法

\`\`\`bash
/req:migrate <req-id>
/req:migrate --all
\`\`\`

## 选项

- \`req-id\` - 需求 ID
- \`--all\` - 批量迁移
- \`--dry-run\` - 预览迁移
- \`--interactive\` - 交互式迁移
- \`--status\` - 查看迁移状态

## 安全特性

- ✅ **自动备份**: 迁移前自动创建 \`.backup/\` 目录
- ✅ **验证检查**: 验证生成的文档完整性
- ✅ **回滚机制**: 支持回滚到迁移前状态

## 输出示例

\`\`\`markdown
🔄 文档迁移 - REQ-20260514-001

📦 备份中...
✓ 创建 .backup/ 目录

📝 生成新文档...
✓ spec.md
✓ test-cases.md

🗑️ 清理旧文档...
✓ 删除旧文档

✅ 迁移完成！
\`\`\`

## 与 /req:unify 的区别

| 特性 | /req:migrate | /req:unify |
| ---- | ------------ | ---------- |
| 备份 | ✅ 自动备份  | ❌ 无备份  |
| 回滚 | ✅ 支持回滚  | ❌ 无回滚  |
| 场景 | 一次性迁移   | 日常统一   |

## 相关命令

- \`/req:unify\` - 快速统一文档格式
- \`/req\` - 创建新需求
- \`/req:quality\` - 质量检查
