# 文档格式统一命令

将需求相关的5个独立文件统一为2个精简文件，消除冗余，降低维护成本。

## 用法

\`\`\`bash
/req:unify <req-id>
/req:unify --all
\`\`\`

## 选项

- \`req-id\` - 需求 ID（如：REQ-20260514-001）
- \`--all\` - 批量统一所有需求
- \`--dry-run\` - 预览模式，不实际修改
- \`--force\` - 强制覆盖已存在的 spec.md

## 优势

- ✅ **维护成本降低 70%**：从5个文件减少到2个
- ✅ **消除信息冗余**：单一信息源
- ✅ **避免不一致**：集中式文档
- ✅ **减少跳转**：所有信息在一个文件

## 输出示例

\`\`\`markdown
🔄 统一文档格式 - REQ-20260514-001

📋 读取旧文档...
✓ design.md
✓ test-plan.md
✓ plan.md
✓ analysis-report.md
✓ CHANGELOG.md

📝 生成新文档...
✓ spec.md (统一主文档)
✓ test-cases.md (详细测试用例)

🗑️ 清理旧文档...
✓ 删除旧文档

✅ 统一完成！
\`\`\`

## 相关命令

- \`/req:migrate\` - 迁移旧格式需求（带备份）
- \`/req\` - 创建新需求（自动使用新格式）
- \`/req:quality\` - 质量检查
